# 示例对话

> 查看实际的用户-AI对话示例

---

## 示例 1：开始新项目

**用户**：
```
/sart
```

**AI（执行 /start 命令后）:**
```
👋 欢迎回到项目

## 📂 项目信息
- **项目名称**: MyBlog
- **技术栈**: Python + Django
- **主要功能**: 个人博客系统

## 🕐 最近动态
feat(post): 完成文章发布功能 - 2小时前

## 📊 当前状态
- 代码状态: ✅ 无未提交修改
- TODO数量: 3个待处理

## 🎯 你可以：
1. 输入 `/progress` - 查看详细进度
2. 输入 `/next` - 获取任务建议
3. 直接说: "继续开发评论功能"
```

```

## 🎯 项目概况
- 项目名称: MyBlog
- 技术栈: Python + Django
- 当前阶段: 开发中
- **整体进度: 65%**

## 📈 功能模块进度

### 后端模块
- 用户认证: 100% ✅
- 文章管理: 100% ✅
- 评论系统: 50% 🚧
- 标签分类: 0% 📋

### 前端页面
- 首页: 100% ✅
- 文章详情: 90% ✅
- 后台管理: 30% 🚧

## 📋 待完成事项
1. [ ] 完成评论系统API
2. [ ] 实现评论审核功能
```

```
看起来评论功能还没完成，接下来我该做什么？
```

**AI**：
```
是的，评论系统目前完成50%。建议接下来：

2. 实现评论审核功能（可选，优先级较低）

```

## 示例 3：获取下一步建议

**用户**：
```
/next
```

**AI（执行 /next 命令后）:**
```
# 🎯 下一步建议

## 📌 当前状态
- 最近在做: 评论系统API开发
- 开发阶段: 中期
- 代码状态: 正常

## 🚀 建议优先级

### 🔴 高优先级（必须马上做）
1. **完成评论系统API**
   - 原因: 基础功能，阻塞前端开发
   - 预计耗时: 2小时
   - 位置: `apis/comments.py`

### 🟡 中优先级（近期要做）
1. **添加评论审核功能**
   - 建议时间: 本周内
   - 位置: `services/comment_service.py`

### 🟢 低优先级（可以考虑）
1. **优化评论查询性能**
   - 说明: 提升列表加载速度
   - 建议时间: 本月内

## 💡 具体行动建议
**建议接下来:**
1. 实现评论的增删改查API
2. 添加权限控制（只有登录用户能评论）
3. 提交代码后运行 `/update-status`
```
