# 工作流示例

> 查看完整的使用场景和实际工作流程

---

## 场景：开发一个新功能（"用户反馈"模块）

### 周一 早上 9:00 - 开始工作

**操作：** 输入 `/start`

**AI 响应:**
```markdown
👋 欢迎回到项目

项目: MyProject
技术栈: Python + Django

最近动态:
feat(user): 完成用户管理模块 - 昨天 17:30

当前状态:
- 代码状态: ✅ 无未提交修改
- TODO数量: 2个

建议:
本周计划开发用户反馈模块
```

---

### 周一 早上 9:05 - 获取任务建议

**操作：** 输入 `/next`

**AI 响应:**
```markdown
🎯 下一步建议

高优先级:
1. 开发用户反馈模块
   - 后端: Model → Service → API
   - 预计耗时: 2天
   - 参考模块: 用户管理

建议接下来:
1. 先设计数据库表结构
2. 完成后运行 /update-status
```

---

### 周一到周二 - 开发中

用户专注于开发...

---

### 周二下午 15:00 - 开发完成，提交代码

```bash
git add .
git commit -m "feat(feedback): 完成用户反馈模块"
git push
```

**操作：** 输入 `/update-status`

**AI 响应:**
```markdown
📝 项目状态更新报告

本次更新:
- ✅ 新增已完成任务: 用户反馈模块
- 📋 新增待办任务: 添加单元测试

进度变化:
- 整体进度: 65% → 72%

文档已更新: docs/项目状态.md
```

---

### 周三 早上 9:00 - 产品经理问进度

**操作：** 输入 `/progress`

**AI 生成报告:**
```markdown
📊 项目进度报告

整体进度: 72%

后端模块:
- 用户管理: 100%
- 支付模块: 100%
- 用户反馈: 100% ✅

待办:
- 添加单元测试（优先级: 中）
```

用户：复制报告发给产品经理

---

## 耗时统计

| 任务 | 耗时 |
|------|------|
| 开始工作 | 1分钟 |

| 生成报告 | 1分钟 |
| **总计** | **4分钟** |

传统方式需要 1-2 小时
