# 自动上下文压缩

## 触发阈值

| 阈值 | 行为 |
|------|------|
| >= 100K tokens | 提醒建议压缩 |
| >= 120K tokens | **自动压缩**（无需确认）|
| >= 150K tokens | **紧急压缩**（保留最少必要信息）|

## 压缩保留（高信号）

- 最近 5 个编辑的文件（完整内容）
- 架构决策和技术选型理由
- 未解决的 Bug（问题描述+当前进展）
- 最近 3 次测试结果（通过/失败统计）
- claude-progress.txt 中的关键信息

## 压缩删除（低信号）

- 旧的工具调用输出（仅保留最近10次调用名称）
- 历史对话过程（保留最近5轮核心结论）
- 完整错误堆栈（保留错误类型+解决方案）

## 自动压缩流程

1. 检测 token 使用量达到阈值
2. 生成压缩摘要（当前任务/最近变更/关键决策/未解决问题/测试状态）
3. 保存摘要到 `~/.claude/context-compression-history/`
4. 汇报压缩结果（压缩前后 token 对比）

## 压缩后恢复（下次会话启动时）

1. 读取 claude-progress.txt
2. 读取 feature_list.json
3. `git log --oneline -5`
4. 加载最近编辑的文件
5. 读取压缩摘要（如果存在）

## 预防措施

- 不要一次性读取整个项目
- 每个任务只读取相关的 3-5 个文件
- 完成一个功能后主动"忘掉"代码细节
- 频繁 commit，即使溢出也不丢失工作

## 汇报格式

```
上下文自动压缩完成
- 压缩前：{原token数} tokens
- 压缩后：{新token数} tokens
- 节省：{百分比}%
- 恢复方法：下次启动自动加载最近5个文件
```
