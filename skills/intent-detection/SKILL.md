# 智能意图识别

## 意图分类（每个请求自动执行）

| 意图 | 信号词 | 自动执行 |
|------|--------|----------|
| Debug | bug, error, 报错, 失败, 不工作, 挂了, 崩了, 异常 | systematic-debugging skill |
| Feature | 新功能, 实现, 添加, 创建, 开发, 写一个, 做一个 | tdd + brainstorming skill |
| Review | 检查, 审查, 看看代码, review, 评审, 代码质量 | code-review skill |
| Refactor | 重构, 优化, 清理, 简化, 改进, 整理 | taste-check skill |
| Commit | 提交, commit, 完成了, 推送, push | git commit workflow |
| Research | 怎么做, 如何, 查一下, 了解, 这个库怎么用 | researcher agent |
| Planning | 计划, 规划, 设计, 架构, 方案, 怎么设计 | create-plan skill |
| Docs | 文档, readme, 注释, 说明 | document-writer agent |

**无法分类** → 当作普通对话，直接回答

## 复杂度评估

| 复杂度 | 判断标准 | 处理方式 |
|--------|----------|----------|
| 简单 | 单文件, <20行, 明确小改动 | 直接执行，不规划 |
| 中等 | 2-5文件, 20-200行, 新增API | 简要思考后执行 |
| 复杂 | 5+文件, 200+行, 涉及架构 | 先深度思考，创建 task_plan.md |

## 自动确认规则

### 自动执行（不问用户）

条件**全部**满足时自动确认为 Y：
- 任务复杂度 T1 或 T2
- 预估行数 < 100
- 无风险关键词
- 用户未明确要求确认

### 强制确认（必须等用户）

**任一**满足即需确认：
- T3/T4 任务
- 包含风险关键词
- 用户明确要求确认
- 膨胀因子 > 3（实际改动远超预估）

## 风险关键词黑名单

数据库, 认证, 支付, 删除, 部署, 迁移, 重构, 架构, 核心, 系统, 基础设施

## 铁律

1. **意图分类是隐式的** — 不要问"你是想 debug 还是 review？"
2. **自动执行，不要建议** — 不要说"建议使用 /debug 命令"
3. **复杂问题先思考** — 复杂度 >= 中等时先输出思考过程
4. **使用 skill 工具** — Debug → `skill("systematic-debugging")`，Feature → `skill("tdd")`

## 示例

### T1 自动执行
```
用户："把变量名从 a 改成 userName"
AI 判断：T1 + 无风险 → 自动确认，直接改
```

### T4 强制确认
```
用户："重构支付系统"
AI 判断：T4 + 风险词"重构""支付" → 显示协议，等待确认
```
