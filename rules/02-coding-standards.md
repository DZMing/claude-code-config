# 代码标准与规范 - 严禁违背

## 语言规范

- **主语言**：所有回复/说明/总结必须使用**简体中文**
- **禁止英文**：除代码、命令、文件路径、日志原文外，不输出英文句子
- **命令输出**：英文输出必须用中文解释关键结论
- **代码注释**：关键逻辑必须写中文注释

## 错误处理

- 外部调用（API/DB/文件）必须 try/catch
- 记录错误上下文（who/what/where），不要裸 `console.error(error)`

## Git Commit 规范

格式：`<类型>(<范围>): <描述> (<状态>)`

| 类型 | 用途 | 强制要求 |
|------|------|----------|
| feat | 新功能 | 关联 feature_list.json |
| fix | 修复 Bug | 关联 Issue 或问题描述 |
| refactor | 重构 | 通过原有测试 |
| perf | 性能优化 | 提供 Benchmark 对比 |
| test | 添加测试 | 说明覆盖功能点 |
| docs | 文档更新 | 无需测试 |
| style | 样式/格式 | 仅 CSS 或格式 |
| chore | 杂项 | 清理、依赖、配置 |

状态标记：`(Pass Test)` / `(Coverage XX%)` / `(No Test)` / `(WIP)` / `(Benchmark: X->Y)`
