# 代码标准与规范

## Git Commit 规范

格式：`<类型>(<范围>): <描述> (<状态>)`

| 类型     | 用途     |
| -------- | -------- |
| feat     | 新功能   |
| fix      | 修复 Bug |
| refactor | 重构     |
| perf     | 性能优化 |
| test     | 添加测试 |
| docs     | 文档更新 |
| chore    | 杂项     |

状态标记：`(Pass Test)` / `(Coverage XX%)` / `(No Test)` / `(WIP)`
